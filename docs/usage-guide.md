# Screenshot-Based Usage Guide

This guide is written as a step-by-step walkthrough you can pair with screenshots from the app.

## 1. Open the app

Capture the main dashboard after the app loads.

What to show in the screenshot:
- The top header with the app title
- The dashboard tabs
- The user chip
- The active dashboard content

What the user should notice:
- The app opens on the Professional Dashboard by default
- The current user is shown in the top right
- The tracker data is already loaded

## 2. Switch between dashboards

Capture the tab row with each of these states:
- Professional Dashboard selected
- Personal Tracker selected
- Insights Dashboard selected

What to show in the screenshot:
- The active tab highlight
- The content area changing between views
- The header staying consistent

What the user should notice:
- Professional Dashboard is for SEO issue tracking
- Personal Tracker is for daily habits and self-management
- Insights Dashboard summarizes activity and progress

## 3. Use the login modal

Capture the login modal in both modes:
- Sign Up
- Login

What to show in the screenshot:
- Email field
- Password field
- Sign Up / Login tabs
- Continue button
- Error message area if needed

What the user should notice:
- Sign Up creates a new Supabase account
- Login uses an existing Supabase account
- The modal appears when switching users or signing out

## 4. Switch user

Capture the app after pressing Switch User.

What to show in the screenshot:
- The user chip changing to Guest
- The login modal visible again
- The dashboards still present behind the modal

What the user should notice:
- Switch User logs out the current session
- The app resets user-specific state
- The login form is shown immediately

## 5. Add a task

Capture the Add Task panel expanded.

What to show in the screenshot:
- Phase selector
- Task name input
- URL / page input
- Effort input
- Day number input
- Add Task button

What the user should notice:
- New SEO tasks can be added directly from the dashboard
- The form is compact on desktop and stacked on mobile

## 6. Update task status

Capture a task row with the status badge visible.

What to show in the screenshot:
- Issue title
- Due date field
- Owner field
- Status badge
- Effort pill
- Notes field

What the user should notice:
- Clicking the status badge cycles the task through its states
- Edits save in the browser automatically

## 7. Mobile view

Capture the app at a narrow phone width.

What to show in the screenshot:
- The stacked top controls
- The modal fitting the screen
- The table becoming horizontally scrollable instead of crushed

What the user should notice:
- The phone layout is optimized with smaller spacing
- The modal and controls fit a narrow screen better
- Tables can be scrolled sideways on mobile
